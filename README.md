## Описание приложения

**StoreProject** — мобильное Android-приложение интернет-магазина одежды. Позволяет просматривать каталог товаров с фильтрацией по категориям, выбирать размер, добавлять позиции в корзину и оформлять заказ. Корзина сохраняется между сессиями в локальной базе данных.

## Реализованный функционал

- **Каталог товаров** — список одежды с изображениями, названиями, описаниями и ценами.
- **Фильтрация по категориям** — Новинки (по тегу), Джинсы, Футболки, Платья, Верхняя одежда, Худи.
- **Офлайн-режим** — каталог кешируется в Room; при отсутствии сети показываются кешированные данные.
- **Детальный просмотр товара** — bottom sheet с фотографией, описанием, ценой, выбором размера и характеристиками.
- **Корзина** — добавление товаров с выбором размера, управление количеством (+/−), удаление позиций, очистка с подтверждением.
- **Персистентность корзины** — содержимое корзины сохраняется в Room DB (только id товара, id размера, количество) и восстанавливается при следующем запуске.
- **Оформление заказа** — форма с валидацией имени и e-mail, кнопка «Оформить» заблокирована до заполнения обязательных полей.
- **Нижняя навигация** — переключение «Меню» / «Корзина», бейдж-счётчик на иконке корзины.

## Архитектурная схема

```
┌─────────────────────────────────────────────────────────────┐
│                        UI Layer                              │
│                                                             │
│  MainActivity                                               │
│    ├── NavHost                                              │
│    │     ├── MainScreen (каталог, фильтры)                  │
│    │     └── CartScreen (корзина, форма заказа)             │
│    └── BottomBar (навигация + бейдж)                        │
│                                                             │
│  Composable-компоненты:                                     │
│    ProductBottomSheet · CartItemRow · ProductItem · ...     │
└──────────────────┬──────────────────────┬───────────────────┘
                   │ observes              │ observes
       ┌───────────▼──────────┐  ┌────────▼──────────┐
       │   CatalogViewModel   │  │   CartViewModel    │
       │  (AndroidViewModel)  │  │   (ViewModel)      │
       │                      │  │                    │
       │  uiState: StateFlow  │  │ items: Map<…,Int>  │
       │  allProducts: SF     │  │ loadFromDb()       │
       └──────────┬───────────┘  └────────┬───────────┘
                  │                        │
       ┌──────────▼───────────┐  ┌────────▼──────────┐
       │  ProductRepository   │  │     CartDao        │
       │  loadFromCache()     │  │  getAll()          │
       │  loadFromNetwork()   │  │  insertAll()       │
       └──────────┬───────────┘  │  deleteAll()       │
                  │               └────────┬───────────┘
       ┌──────────┴──────────────────────┐ │
       │          Data Layer              │ │
       │  ┌────────────┐  ┌───────────┐  │ │
       │  │ CatalogDao │  │  Room DB  │◄─┘ │
       │  │ (products, │  │ catalog_db│    │
       │  │ categories)│  │           │◄───┘
       │  └────────────┘  │  tables:  │
       │                  │  products │
       │  ┌────────────┐  │  categories│
       │  │ Retrofit   │  │  cart_items│
       │  │ CatalogApi │  └───────────┘
       │  └────────────┘
       └─────────────────────────────────┘
```

**Поток данных:**
1. `CatalogViewModel.loadCatalog()` сначала читает кеш из Room, затем загружает из сети и обновляет кеш.
2. `CartViewModel.loadFromDb()` вызывается однократно после загрузки каталога; восстанавливает корзину, объединяя записи БД с доменными объектами.
3. Любое изменение корзины синхронно обновляет in-memory `Map` (реактивный UI) и асинхронно сохраняет в Room через `syncToDb()`.

## Инструкция по сборке

### Требования

- Android Studio Hedgehog (2023.1) или новее
- JDK 11+
- Android SDK с установленными:
  - Build Tools 34+
  - Platform SDK API 36

### Шаги

```bash
# 1. Клонировать репозиторий
git clone <repo-url>
cd team-daiquiri

# 2. Открыть в Android Studio
# File → Open → выбрать папку team-daiquiri

# 3. Собрать debug-APK из командной строки
./gradlew assembleDebug

# APK будет по пути:
# app/build/outputs/apk/debug/app-debug.apk

# 4. Запустить тесты
./gradlew test

# 5. Проверить линтер
./gradlew ktlintCheck
```

### Конфигурация сети

API-токен захардкожен в `RetrofitInstance.kt`. При необходимости замените его перед сборкой.
